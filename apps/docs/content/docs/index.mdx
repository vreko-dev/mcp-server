## How It Works: Learning Loop

### Day 1: Accurate Out of the Box

You install SnapBack. It detects hardcoded secrets, phantom dependencies, test code in production, and risky patterns. 94% accuracy on the first day.

### Day 30: Knows Your Codebase

Every time you restore files, SnapBack learns what patterns caused the problem. Over 30 days, it figures out YOUR specific risks:
- "Cursor always breaks this auth module"
- "This refactor pattern causes cascading failures"
- "This folder has phantom dependency issues"

Accuracy improves to 98%.

### Month 3: Catches What Others Miss

After 90 days, SnapBack has learned what breaks in YOUR specific codebase. It catches patterns before you even think to look for them. It knows:
- Folder-specific risks (which folders are historically fragile)
- Pattern-specific risks (which refactoring patterns cause issues)
- Tool-specific risks (which AI tool is reckless with which files)

Accuracy reaches 99%+.

## Features

- **Pattern Memory** - Learns what went wrong in YOUR codebase
- **Progressive Learning** - Gets smarter: Day 1 (94%) → Day 30 (98%) → Month 3 (99%+)
- **Detects What Breaks** - Hardcoded secrets, phantom deps, test code in prod, risky patterns
- **Multi-File Restore** - Undo entire features, not just individual files
- **Per-Tool Learning** - Learns which AI tools break which parts of your code
- **Privacy First** - Metadata-only, your code stays local
