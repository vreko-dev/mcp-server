## Migration Strategy

### Phase 2A: Update Constructor Definitions (result.ts)

**File:** `apps/vscode/src/types/result.ts`

**Current:**
```typescript
export function Ok<T>(value: T): Result<T, never> {
  return { success: true, value };
}

export function Err<E = Error>(error: E): Result<never, E> {
  return { success: false, error };
}
```

**Target:**
```typescript
// New lowercase constructors (canonical)
export function ok<T>(value: T): Result<T, never> {
  return { success: true, value };
}

export function err<E = Error>(error: E): Result<never, E> {
  return { success: false, error };
}

// Deprecated uppercase aliases (backward compatibility)
/**
 * @deprecated Use lowercase `ok()` instead. Will be removed in v2.0.
 */
export const Ok = ok;

/**
 * @deprecated Use lowercase `err()` instead. Will be removed in v2.0.
 */
export const Err = err;
```

**Impact:**
- ✅ Zero breaking changes (aliases maintain compatibility)
- ✅ TypeScript shows deprecation warnings
- ✅ Codemod can safely migrate callsites
