# Usage Components

The usage components provide a consistent way to display usage information and limits to users in the SnapBack dashboard. These components follow the Resource pattern for consistent state management.

## Components

### UsageSection

The main component that displays usage information including limits, warnings, and subscription details.

```tsx
import { UsageSection } from "@/modules/saas/usage/components/UsageSection";

<UsageSection limits={limits} subscription={subscription} />;
```

#### Props

-   `limits`: Usage limits data
-   `subscription`: Subscription information

#### Companion States

The UsageSection component includes companion states for all Resource pattern states:

-   `UsageSection.Skeleton` - Loading skeleton
-   `UsageSection.Empty` - Empty state
-   `UsageSection.Error` - Error state

### UsageProgressBar

Displays a progress bar showing usage against limits.

```tsx
import { UsageProgressBar } from "@/modules/saas/usage/components/UsageProgressBar";

<UsageProgressBar
	used={limits.snapshotsUsed}
	limit={limits.snapshotsLimit}
	label="Checkpoints this month"
	unit="checkpoints"
/>;
```

#### Props

-   `used`: Amount used
-   `limit`: Usage limit
-   `label`: Label for the progress bar
-   `unit`: Unit of measurement

#### Companion States

-   `UsageProgressBar.Skeleton` - Loading skeleton
-   `UsageProgressBar.Empty` - Empty state
-   `UsageProgressBar.Error` - Error state

### UsageWarningBanner

Displays warnings when usage approaches limits.

```tsx
import { UsageWarningBanner } from "@/modules/saas/usage/components/UsageWarningBanner";

<UsageWarningBanner
	percentage={checkpointPercentage}
	resourceType="checkpoints"
	upgradeUrl="/choose-plan"
/>;
```

#### Props

-   `percentage`: Usage percentage
-   `resourceType`: Type of resource (e.g., "checkpoints", "storage")
-   `upgradeUrl`: URL to upgrade page (optional)

#### Companion States

-   `UsageWarningBanner.Skeleton` - Loading skeleton
-   `UsageWarningBanner.Empty` - Empty state (returns null)
-   `UsageWarningBanner.Error` - Error state

### UpgradeModal

Modal that prompts users to upgrade when they reach usage limits.

```tsx
import { UpgradeModal } from "@/modules/saas/usage/components/UpgradeModal";

<UpgradeModal
	isOpen={showUpgradeModal}
	onClose={() => setShowUpgradeModal(false)}
	resourceType="checkpoints"
	currentPlan={subscription.plan}
/>;
```

#### Props

-   `isOpen`: Whether the modal is open
-   `onClose`: Function to close the modal
-   `resourceType`: Type of resource
-   `currentPlan`: Current subscription plan

#### Companion States

-   `UpgradeModal.Skeleton` - Loading skeleton
-   `UpgradeModal.Empty` - Empty state (returns null)
-   `UpgradeModal.Error` - Error state

## Hooks

### useUsageLimits

Hook to fetch usage limits data with Resource pattern.

```tsx
import { useUsageLimits } from "@/hooks/use-usage";

const usageLimitsR = useUsageLimits();
```

### useSubscriptionInfo

Hook to fetch subscription information with Resource pattern.

```tsx
import { useSubscriptionInfo } from "@/hooks/use-usage";

const subscriptionInfoR = useSubscriptionInfo();
```

## Usage Example

```tsx
"use client";

import { matchResource } from "@/lib/resource";
import { useUsageLimits, useSubscriptionInfo } from "@/hooks/use-usage";
import { UsageSection } from "@/modules/saas/usage/components/UsageSection";

export function UsageDashboard() {
	const usageLimitsR = useUsageLimits();
	const subscriptionInfoR = useSubscriptionInfo();

	return matchResource(usageLimitsR, {
		loading: () => <UsageSection.Skeleton />,
		empty: () => <UsageSection.Empty />,
		error: (error) => <UsageSection.Error error={error} />,
		ready: (limits) =>
			matchResource(subscriptionInfoR, {
				loading: () => <UsageSection.Skeleton />,
				empty: () => <UsageSection.Empty />,
				error: (error) => <UsageSection.Error error={error} />,
				ready: (subscription) => (
					<UsageSection limits={limits} subscription={subscription} />
				),
			}),
	});
}
```
