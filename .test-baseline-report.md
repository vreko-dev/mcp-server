## Root Cause Analysis

### Primary Issue: Async Error Handling Bug

**Location:** `SessionCoordinator.ts:192-221`

```typescript
async finalizeSession(reason: SessionFinalizeReason): Promise<SessionId | null> {
    // ... session creation logic ...
    
    try {
        // Store the session manifest
        await this.storeSessionManifest(manifest); // Line 194
        
        // ... event emission and logging ...
        
        return sessionId;
    } catch (error) {
        this.logger.error(...); // Line 213
        return null;           // ❌ Error swallowed - no storage
    }
}
```

**The Problem:**
1. `storeSessionManifest` re-throws errors after logging (line 341: `throw error;`)
2. `finalizeSession` catches and swallows these errors, returning `null`
3. Mock storage never receives the manifest when errors occur
4. Tests expect `mockStorage.storedManifests[0]` but array is empty

**Why Tests Fail:**
```typescript
// Test expectation:
expect(mockStorage.storedManifests[0].reason).toBe("manual");
//     ^^^^^^^^^^^^^^^^^^^^^^^^^^^^ undefined

// Actual behavior:
// - finalizeSession returns null (caught error)
// - storedManifests array is empty []
// - Test tries to access [0].reason on undefined
```

## Deeper Investigation Required

### Question 1: Why is `storeSessionManifest` throwing?

The mock implementation looks correct:
```typescript
class MockSessionStorage implements ISessionStorage {
    public storedManifests: SessionManifest[] = [];
    public shouldFail = false;

    async storeSessionManifest(manifest: SessionManifest): Promise<void> {
        if (this.shouldFail) {
            throw new Error("Storage failure");
        }
        this.storedManifests.push(manifest); // Should succeed
    }
}
```

**Hypothesis:** There may be a validation error or type mismatch in the `manifest` object being passed.

### Question 2: Is the Session Too Short?

Line 167 guards against short sessions:
```typescript
if (sessionDuration < this.config.minSessionDuration || this.candidates.size === 0) {
    this.resetSession();
    return null;
}
```

**Default:** `minSessionDuration = 5000ms` (5 seconds from THRESHOLDS)

**Test Setup:** Tests don't wait 5 seconds, so sessions are created instantly.

**Verdict:** This is likely the primary cause of all failures.

## Recommended Fixes

### Option A: Mock Time in Tests ⭐ (Recommended)

Update test setup to advance time past minimum duration:

```typescript
beforeEach(() => {
    mockEventEmitter = new MockEventEmitter();
    mockTimers = new MockTimerService();
    mockLogger = new MockLogger();
    mockStorage = new MockSessionStorage();

    // Mock Date.now() to control time
    vi.useFakeTimers();
    vi.setSystemTime(new Date('2025-01-01T00:00:00Z'));

    coordinator = new SessionCoordinator({
        storage: mockStorage,
        timers: mockTimers,
        logger: mockLogger,
        eventEmitter: mockEventEmitter,
        config: { minSessionDuration: 0 }, // ⭐ Override to 0ms for tests
    });
});

afterEach(() => {
    if (coordinator) {
        coordinator.dispose();
    }
    vi.useRealTimers();
});
```

**Impact:**
- ✅ Minimal code change
- ✅ Tests fast (no real waiting)
- ✅ Keeps production logic intact
- ❌ Need to update all 11 tests

## Decision: Recommended Approach

**Use Option B** with supplementary tests for time-based validation:

1. **Default Test Setup:** `minSessionDuration: 0` for fast unit tests
2. **Add Dedicated Tests:** Validate 5s minimum with fake timers:
   ```typescript
   describe("Session Duration Validation", () => {
       it("should reject sessions shorter than minSessionDuration", async () => {
           const shortCoordinator = new SessionCoordinator({
               storage: mockStorage,
               config: { minSessionDuration: 5000 },
           });
           shortCoordinator.addCandidate("file:///test.ts", "snapshot-123");
           // Don't wait

       });
   });
   ```

**Rationale:**
- ✅ Fast test execution (no waiting)
- ✅ Minimal code change
- ✅ Time validation tested separately
- ✅ All 11 failures fixed with one config line
