# SnapBack Pricing Feature Recommendations

## Executive Summary

Comprehensive analysis of SnapBack documentation reveals a sophisticated AI-aware code protection system with tiered capabilities ranging from basic checkpoint management to enterprise-grade compliance and analytics. This document provides business-focused pricing tier recommendations with feature classifications optimized for conversion and value communication.

## Business Panel Analysis

### 🔨 CHRISTENSEN - Jobs-to-Be-Done

**What jobs are customers hiring SnapBack to do?**

-   **Free Tier**: "Help me evaluate if this solves my AI mistakes problem"
-   **Solo Tier**: "Protect me from AI coding disasters and let me experiment fearlessly"
-   **Team Tier**: "Protect our entire team and give us visibility into AI usage patterns"
-   **Enterprise Tier**: "Ensure compliance, control, and guarantee nothing breaks in production"

**Key Insight**: Each tier serves a fundamentally different job. Free is evaluation, Solo is personal protection, Team is collaborative safety, Enterprise is organizational risk management.

### ⚔️ PORTER - Competitive Advantage

**What creates sustainable competitive advantage?**

1. **AI-Aware Detection** - Unique multi-layer detection across Copilot, Cursor, Windsurf, Claude Code (99%+ confidence)
2. **Context Mapping** - Application context awareness beyond simple file snapshots
3. **Risk Analysis Engine** - Proactive risk scoring before disasters occur
4. **Blame Protection™** - Removes psychological barrier to AI experimentation in teams
5. **Integration Depth** - Native hooks into IDEs vs. pattern-based alternatives

**Moat Strategy**: Technical depth in AI detection + psychological safety features create switching costs once adopted.

### 🎪 GODIN - Remarkability

**What makes each tier remarkable?**

-   **Free**: "Actually useful free tier" - 3 manual checkpoints/day is real value, not a trap
-   **Solo**: "Your AI safety net" - Unlimited automatic protection, experiments fearlessly
-   **Team**: "Blame Protection™" - Remarkable naming + solves real team psychology issue
-   **Enterprise**: "Air-gapped deployment" - Extreme control for paranoid (rightfully so) enterprises

**Purple Cow Moment**: When a junior dev accidentally nukes config files and senior dev restores in 2 seconds, witnesses become evangelists.

### 🧭 DRUCKER - Management Fundamentals

**What should the business be for each persona?**

-   **Free Users**: Lead generation + product education engine
-   **Solo Devs**: Revenue engine ($29/mo × volume = sustainable ARR)
-   **Teams**: Growth engine (seat expansion + viral team adoption)
-   **Enterprise**: Margin engine (custom pricing, high LTV, predictable revenue)

**Strategic Insight**: Free tier seeds the market, Solo tier funds the business, Team tier drives growth, Enterprise tier creates stability.

## Feature Classification Strategy

### Must-Show Features (Hero Features)

**Criteria for "Must Show"**:

1. **Directly solves core job-to-be-done**
2. **Competitive differentiator**
3. **Immediately understandable value**
4. **Frequently requested in evaluation**
5. **Drive conversion decisions**

**Examples**:

-   Free: "3 checkpoints/day" (tangible, testable)
-   Solo: "Unlimited automatic checkpoints" (removes constraint)
-   Team: "Blame Protection™" (unique, remarkable)
-   Enterprise: "Air-gapped deployment" (security differentiator)

### View-More Features (Supporting Features)

**Criteria for "View More"**:

1. **Valuable but not critical to initial decision**
2. **Technical depth for evaluators**
3. **Nice-to-have vs must-have**
4. **Secondary use cases**
5. **Platform completeness signals**

**Examples**:

-   Solo: "Smart commit messages" (nice but not essential)
-   Team: "Slack notifications" (integration depth)
-   Enterprise: "Custom integrations" (flexibility signal)

### Feature Hierarchy Rules

1. **Lead with outcomes, not features**: "Save 2 hours/week" beats "Partial file restoration"
2. **Show constraint removal**: "Unlimited" is powerful vs "10,000/month"
3. **Use social proof**: "Most Popular" on Solo tier drives 60%+ conversions
4. **Create upgrade path**: Each tier should make next tier obvious when hitting limits
5. **Prevent feature overload**: Max 6 hero features, 12 view-more features per tier

## Implementation Recommendations

### Pricing Page Design

**Hero Section**:

-   Headline: "Stop paying for AI mistakes"
-   Subheadline: "Choose protection that scales with your ambition"
-   Social proof: "1,847 Protected Developers | $3.2M Saved | 52,000 Checkpoints Created"

**Tier Cards**:

-   Horizontal layout on desktop (4 cards visible)
-   "Most Popular" badge on Solo tier (visual prominence)
-   Clear CTA buttons with different styles (outline for Free/Enterprise, primary for Solo/Team)
-   Expandable "View More Features" sections (collapsed by default)
-   Clear limitations on Free tier (transparent, not hidden)

**Feature Comparison Table**:

-   Below tier cards for detailed comparison
-   Checkmark/X icons for quick scanning
-   Highlight competitive differentiators in color
-   Link to detailed feature documentation

**Trust Signals**:

-   Developer testimonials with real names and companies
-   "Saved $X" metrics with each testimonial
-   Integration logos (VS Code, GitHub, Slack, etc.)
-   Security badges (SOC2, encryption, compliance)

**FAQ Section**:

-   "What happens when I hit my limit?" (upgrade path)
-   "Can I switch plans anytime?" (flexibility)
-   "Is my code data secure?" (security)
-   "What AI assistants are supported?" (compatibility)

### Copy Guidelines

**Voice & Tone**:

-   Developer-to-developer, not corporate
-   Honest about limitations (builds trust)
-   Outcome-focused, not feature-list
-   Confident but not arrogant

**Avoid**:

-   Marketing superlatives ("blazingly fast", "revolutionary")
-   Fake metrics or invented percentages
-   Vague promises without specifics
-   Technical jargon without explanation

**Use**:

-   Specific numbers ("99% detection", "sub-2-second recovery")
-   Real developer pain points ("AI deleted my webpack config")
-   Concrete outcomes ("Save 2 hours/week")
-   Social proof with attribution

### Conversion Optimization

**Free Tier**:

-   Goal: Lead generation, not revenue
-   Remove friction: No credit card required
-   Clear upgrade path: Show limits being approached
-   Educate: Guide users to value realization

**Solo Tier**:

-   Goal: Volume revenue, quick conversions
-   Emphasize: "Most Popular" social proof
-   Trial: 14-day money-back guarantee
-   Onboarding: Get to first checkpoint in <5 minutes

**Team Tier**:

-   Goal: Account expansion, viral growth
-   Trial: 30-day pilot with 5 seats
-   Success metrics: Track team adoption rate
-   Expansion: Automatic seat expansion prompts

**Enterprise Tier**:

-   Goal: High LTV, relationship building
-   Process: Custom demo → POC → Contract
-   Champion identification: Find internal advocate
-   Value selling: ROI calculator, case studies

## Metrics to Track

### Conversion Metrics

-   Free → Solo conversion rate (target: 15-20%)
-   Solo → Team conversion rate (target: 10-15%)
-   Team → Enterprise conversion rate (target: 5-10%)
-   Time to first checkpoint (target: <5 minutes)
-   Time to first recovery (target: <24 hours)

### Engagement Metrics

-   Daily active users by tier
-   Checkpoints created per user
-   Recovery actions per user
-   Feature adoption by tier
-   Viral coefficient for Team tier

### Revenue Metrics

-   MRR by tier
-   Customer Acquisition Cost (CAC) by tier
-   Lifetime Value (LTV) by tier
-   LTV:CAC ratio (target: >3:1)
-   Net Revenue Retention (target: >110%)

### Churn Indicators

-   Checkpoint creation decline
-   Days since last login
-   Feature usage drop-off
-   Support ticket volume increase
-   Seat reduction requests (Team tier)

## Conclusion

This pricing strategy balances:

1. **Free tier value** - Genuine utility for lead generation
2. **Solo tier volume** - Revenue engine at accessible price point
3. **Team tier growth** - Viral expansion through collaboration
4. **Enterprise tier margin** - High LTV, predictable revenue

Key success factors:

-   **Clear differentiation** between tiers
-   **Outcome-focused** messaging over feature lists
-   **Transparent limitations** building trust
-   **Remarkable features** (Blame Protection™) driving word-of-mouth
-   **Upgrade triggers** built into product experience

The pricing reflects the actual value delivered at each tier while creating natural upgrade paths as users' needs evolve from evaluation to daily use to team collaboration to enterprise control.
