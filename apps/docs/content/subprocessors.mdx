# Subprocessors

SnapBack uses the following third-party subprocessors to deliver our service. This list is maintained for transparency and compliance purposes (e.g., GDPR Article 28).

## Active Subprocessors

### AWS (Amazon Web Services)
- **Service Purpose**: Cloud infrastructure and snapshot storage
- **Data Processed**: Cloud backup snapshots (metadata only, encrypted), Guardian detection results
- **Location**: US-East-1 region (Virginia, USA)
- **DPA Status**: AWS Customer Agreement includes DPA terms

### Vercel
- **Service Purpose**: Web hosting and edge functions
- **Data Processed**: Web application hosting, authentication sessions
- **Location**: Global CDN (primary: US regions)
- **DPA Status**: Vercel Terms of Service include data processing terms

### PostHog (Cloud, US)
- **Service Purpose**: Product analytics (event names, device/page metadata)
- **Data Processed**: Anonymous usage events, feature adoption metrics. No code content.
- **Location**: US region (cloud-hosted instance)
- **DPA Status**: DPA available on request from PostHog

### Supabase
- **Service Purpose**: PostgreSQL database and authentication
- **Data Processed**: User accounts, subscription data, team memberships
- **Location**: US-East-1 region (Virginia, USA)
- **DPA Status**: Supabase Terms of Service include GDPR-compliant data processing terms

## What Data is Shared?

### With AWS
- **Cloud Backups**: Snapshot metadata (file paths as hashes, timestamps, sizes)
- **Guardian Results**: Detection types (e.g., "secret detected") without actual content
- **Encryption**: All data encrypted at rest (AES-256) and in transit (TLS 1.3)

### With Vercel
- **Web Sessions**: Authentication tokens, session metadata
- **No Code**: Vercel never processes snapshot content

### With PostHog
- **Usage Events**: Feature usage, page views, button clicks
- **No PII**: Events are anonymized; no email addresses or names
- **No Code**: PostHog never receives code content or snapshot data

### With Supabase
- **User Data**: Email, name, subscription status, team memberships
- **No Code**: Supabase never stores snapshot content

## What is NOT Shared?

**Never shared with any subprocessor:**
- Your actual code content
- Sensitive data values (API keys, passwords, tokens)
- Private file contents
- Personal information beyond necessary account data

## Subprocessor Changes

We will notify Enterprise customers at least 30 days in advance of adding new subprocessors or changing data processing locations.

**Notification method:**
- Email to primary account contact
- Notice in Enterprise admin dashboard
- Update to this page (with change log)

The following subprocessors will be added when Enterprise SSO features launch:

- **Okta** (if using Okta SSO): Identity verification
- **Azure AD** (if using Microsoft SSO): Identity verification
- **Google Workspace** (if using Google SSO): Identity verification

**Note**: Only the SSO provider you choose will be added as a subprocessor.

## Data Processing Agreements

### For Enterprise Customers

Enterprise customers can request a Data Processing Agreement (DPA) that covers:
- SnapBack's data processing commitments
- Subprocessor disclosure and approval
- Data transfer mechanisms (Standard Contractual Clauses if needed)
- Security measures and incident response

Contact sales@snapback.dev to request a DPA.

### Subprocessor DPAs

We maintain DPAs with all critical subprocessors:
- ✅ AWS: Covered by AWS Customer Agreement
- ✅ Vercel: Covered by Vercel Terms of Service
- ✅ PostHog: DPA available on request
- ✅ Supabase: Covered by Supabase Terms of Service

## Compliance

All subprocessors are selected based on:
- **GDPR Compliance**: GDPR-compliant data handling practices
- **Security Standards**: SOC 2 Type II or equivalent certification
- **Data Encryption**: Encryption at rest and in transit
- **Incident Response**: Documented breach notification procedures
