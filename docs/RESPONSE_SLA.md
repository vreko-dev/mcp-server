# Community Response Expectations

At SnapBack, we value our community and strive to be responsive. However, as an open-source project with a small core team, we prioritize based on severity and impact.

## Response SLA (Service Level Agreement)

These are our targets for initial response times. They are goals, not guarantees.

| Category | Priority | Initial Response Target |
| :--- | :--- | :--- |
| **Critical Bug** | P0 (Blocker, Data Loss) | **24 Hours** |
| **Major Bug** | P1 (Broken Feature) | **48 Hours** |
| **Minor Bug** | P2 (Annoyance) | **1 Week** |
| **Feature Request** | - | **2 Weeks** (Triage) |
| **Pull Request** | - | **3-5 Days** (Review) |
| **Discussion** | - | **Best Effort** |

## Escalation

If you have a critical issue that hasn't received a response within the target time, please:

1.  Check if it's a known issue.
2.  Ping us on Discord in the `#help` channel.
3.  Avoid tagging individual maintainers directly.

## How to Help

You can help us meet these targets by:

*   Using the correct [Issue Templates](../.github/ISSUE_TEMPLATE/).
*   Providing clear reproduction steps.
*   Checking the documentation before asking.
*   Helping others in the community!
