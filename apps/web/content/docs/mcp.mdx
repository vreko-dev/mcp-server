## What is MCP?

The Model Context Protocol (MCP) is an open standard that allows applications to share context with AI models and coding assistants. SnapBack leverages MCP to provide AI assistants with information about your codebase protection levels, recent changes, and risk analysis.

## Configuration

### Enabling MCP Integration

MCP integration is enabled by default in SnapBack. To configure specific settings:

```json
// .snapbackconfig
{
  "mcp": {
    "enabled": true,
    "contextDepth": "full", // full, minimal, none
    "shareProtectionLevels": true,
    "shareSessionContext": true,
    "shareAiDetection": true
  }
}
```

### VS Code Configuration

In VS Code, you can control MCP integration through settings:

```json
// VS Code settings.json
{
  "snapback.mcp.enabled": true,
  "snapback.mcp.contextDepth": "full",
  "snapback.mcp.shareProtectionLevels": true,
  "snapback.mcp.shareSessionContext": true
}
```

## Context Sharing

SnapBack shares several types of context through MCP:

### Protection Levels

File protection levels help AI assistants understand which files require extra caution:

```javascript
// Example: AI assistant receives context that .env files are blocked
// The assistant can then avoid suggesting changes to these files
// or at least warn the user about potential risks
```

### Session Context

Current session information helps AI assistants provide more relevant assistance:

```json
{
  "currentSession": {
    "name": "Implement user authentication",
    "duration": "45 minutes",
    "filesModified": ["src/auth.js", "src/config.js", "src/routes.js"],
    "goal": "Add JWT-based authentication to the application"
  }
}
```

### AI Detection Results

Recent AI detection results help assistants avoid introducing similar risks:

```json
{
  "recentRisks": [
    {
      "type": "secret",
      "file": "src/config.js",
      "timestamp": "2023-01-01T09:30:00Z",
      "resolved": true
    }
  ]
}
```

## Privacy and Security

SnapBack's MCP integration prioritizes your privacy and security:

<div className="callout my-6">
  <strong>🔒 Privacy First:</strong> SnapBack only shares metadata through MCP, never your actual code content. All context sharing happens locally on your machine.
</div>

### What's Shared

- File paths (hashed for privacy)
- Protection levels
- Session metadata
- AI detection results (types, not content)
- Configuration settings

### What's Never Shared

- Actual file contents
- Sensitive data values
- Personal information
- Code snippets

## Troubleshooting

### Common Issues

1. **MCP context not appearing**: Verify that MCP integration is enabled in both SnapBack and your AI assistant
2. **Outdated context**: Restart your AI assistant to refresh the context
3. **Privacy concerns**: Adjust context sharing settings to limit what's shared

### Debugging MCP Integration

```bash
# Check MCP status
snapback mcp status

# View shared context
snapback mcp context --format json

# Enable MCP debug logging
snapback --log-level debug mcp status
```
